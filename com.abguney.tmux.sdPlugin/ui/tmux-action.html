i Choose which terminal multiplexer you use. Select psmux if you're on Windows, or tmux for macOS/Linux. This is auto-detected based on your OS.
i CLI Command runs commands directly and is the most reliable method. Keystroke Simulation sends keyboard shortcuts to the focused terminal window.
CLI Settings
i Override the default command for this button. For example, enter split-window -v to split vertically. Leave empty to use the built-in default.
i Full path to the tmux or psmux program. Only needed if it isn't found automatically on your system.
i Path to a specific tmux socket file. Only needed for advanced setups with multiple tmux servers. Most users can leave this empty.
i The name of the session to send commands to. For psmux, this is auto-detected from your last active session. Set manually if you have multiple sessions.
i Enable this if tmux runs inside Windows Subsystem for Linux (WSL) instead of natively on Windows. Commands will be prefixed with wsl.
Keystroke Settings
i The tmux prefix key combination sent before each shortcut. Default is Ctrl+B. Change this if you've remapped your prefix (e.g. to Ctrl+A).
